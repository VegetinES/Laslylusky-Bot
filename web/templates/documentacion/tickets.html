Sistema de Tickets - Guía Completa

Descripción General

El sistema de tickets permite a los usuarios crear conversaciones privadas con el equipo de soporte de tu servidor de manera organizada y eficiente. Utilizando hilos privados de Discord, los usuarios pueden abrir tickets para solicitar ayuda, reportar problemas o cualquier otro motivo que hayas configurado.

Características Principales

  • Hilos privados: Utiliza el sistema nativo de hilos de Discord para mantener las conversaciones organizadas
  • Personalización completa: Configura múltiples tipos de tickets con mensajes personalizados
  • Control de acceso: Define permisos específicos para roles y usuarios individuales
  • Registros detallados: Seguimiento completo de todas las acciones realizadas en el sistema
  • Interfaz intuitiva: Fácil de usar tanto para administradores como para usuarios regulares

Comandos de Configuración

  • /config tickets - Abre el menú principal de configuración de tickets

Proceso de Configuración

Paso 1: Crear un Nuevo Ticket

Desde el menú principal, selecciona "Gestionar Tickets" y luego "Crear Nuevo Ticket". Deberás seleccionar:

  • Canal para mostrar el botón de tickets
  • Canal para los registros (logs) de actividad

Paso 2: Configurar Permisos

Define quién puede gestionar o simplemente ver los tickets:

  • Permisos de Gestión: Permiten control total sobre tickets (obligatorio configurar al menos un rol o usuario)
  • Permisos de Visualización: Permiten solo ver tickets sin interactuar

Paso 3: Personalizar Mensajes

Configura dos tipos de mensajes:

  • Mensaje para abrir tickets: El que contiene los botones que ven los usuarios
  • Mensaje de ticket abierto: El que se muestra dentro del ticket cuando se crea

Puedes personalizar títulos, descripciones, colores, imágenes y más.

Paso 4: Configurar Botones

Añade hasta 5 botones diferentes para distintos tipos de tickets:

  • Personaliza textos, emojis y estilos
  • Define formatos de nombres únicos para cada tipo
  • Cada botón puede tener su propio mensaje de respuesta

Paso 5: Guardar y Desplegar

Al guardar la configuración, el sistema automáticamente:

  • Configura los permisos del canal
  • Envía el mensaje con los botones al canal seleccionado
  • Activa el sistema para que los usuarios puedan crear tickets

Formato de Nombres de Tickets

Personaliza el formato de nombres usando estas variables:

  • {id}: Número único autoincremental por tipo de ticket
  • {userid}: ID del usuario que abre el ticket
  • {usertag}: Nombre de usuario en Discord

Ejemplo: soporte-{id}-{usertag} resultaría en "soporte-1-Usuario#1234"

Interacción con Tickets

Para Usuarios

  • Hacer clic en el botón correspondiente para abrir un ticket
  • El ticket se crea como un hilo privado donde solo pueden ver y participar el creador y el equipo
  • Reciben notificaciones cuando el personal responde a su ticket

Para el Equipo de Soporte

  • Añadir miembros: Pueden añadir usuarios adicionales al ticket
  • Eliminar miembros: Pueden remover usuarios del ticket
  • Archivar tickets: Al finalizar, pueden cerrar el ticket
  • Reabrir tickets: Los tickets archivados pueden reabrirse si es necesario

Consejos de Uso

  • Configura diferentes tipos de tickets para distintas necesidades (soporte técnico, reportes, sugerencias)
  • Utiliza colores distintos para cada tipo de ticket para identificarlos fácilmente
  • Configura mensajes claros que expliquen el propósito de cada tipo de ticket
  • Revisa los logs periódicamente para monitorear la actividad
  • Asigna permisos de gestión solo a miembros de confianza
